# Deployment Guide

This guide walks through deploying the FinScope AI platform using **Supabase** (Database), **Vercel** (Frontend + Backend), and **GitHub Actions** (Cron Jobs). 

This architecture guarantees zero hosting costs and requires no credit card verification.

---

### Step 1: Database Setup (Supabase)

1. Create a free account at [Supabase](https://supabase.com/).
2. Create a new Project (e.g., "finscope-db").
3. Go to **Project Settings -> Database**.
4. Scroll down to **Connection pooling**.
5. Copy the **Connection string**. It should look like this:
   `postgres://[user]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres`
   *Make sure you are using the pooled port `6543` and transaction mode to avoid exhausting connections in Vercel's serverless environment.*

#### Run Migrations and Seed Data locally
Before deploying, you must apply your database schema to Supabase:
1. Clone this repository locally.
2. In the `backend` folder, set `DATABASE_URL` in `.env` to your Supabase string.
3. Run `alembic upgrade head`.
4. Run `python -m app.db.seed`.

---

### Step 2: Vercel Project Setup (Frontend + Backend API)

1. Create a free account at [Vercel](https://vercel.com).
2. Click **Add New -> Project**.
3. Import your GitHub repository (`DEEPA-356/FinScope-AI`).
4. Keep the Framework Preset as **Vite** (Vercel will auto-detect the frontend).
5. Open the **Environment Variables** section and add:
   - `DATABASE_URL`: *(Your Supabase Connection String)*
   - `SECRET_KEY`: *(A random 32+ character string for JWT signing)*
6. Click **Deploy**. 

Vercel will automatically read `vercel.json` and build your React frontend as static assets, and your FastAPI backend (`api/index.py`) as serverless Python functions, all on the same domain!

---

### Step 3: GitHub Actions Setup (Nightly Jobs)

We use GitHub Actions to run the ML jobs (feature engineering and forecasting) that keep user dashboards up to date.

1. Go to your repository on **GitHub**.
2. Go to **Settings -> Secrets and variables -> Actions**.
3. Click **New repository secret**.
4. Add the following secrets:
   - `VERCEL_BACKEND_URL`: The URL Vercel gave you (e.g., `https://finscope-ai.vercel.app`)
   - `INTERNAL_JOB_SECRET`: A secure random password you make up (e.g., `SuperSecretCronToken123`).
5. **CRITICAL**: Go back to your **Vercel** dashboard, add `INTERNAL_JOB_SECRET` to your project Environment Variables, and re-deploy so the backend knows what token to expect.

#### Verification Checklist
- [ ] Ensure no Docker containers are running locally or on external servers.
- [ ] Verify you can log in to your Vercel deployment URL using the test users generated by the seed script.
- [ ] Go to the GitHub Actions tab, select "Nightly ML Jobs", and click **Run workflow** manually to verify the cron jobs execute correctly without hitting the Vercel 10s timeout limit.
